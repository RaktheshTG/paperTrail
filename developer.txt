Basics :
Vite vs React - vite is the tool that runs react during development and execution stage.
                react is a js library that helps in building the ui.


index.html
touch index.html to change the page title, add a favicon, or add a Google Fonts link. 
That's it. it has div id = "root" where react mounts


TanStack Router
TanStack Router is a library that fakes page navigation inside React.
When you go to /about, it doesn't load a new file — it just swaps which
component is displayed. This is called "client-side routing".


router.tsx 
creates the router instance and tells it "here are all the routes in this app."


routes/ folder — each file in here IS a page. The file name = the URL path:

routes/index.tsx → shows at /
routes/about.tsx → would show at /about
routes/__root.tsx → wraps EVERY page (good for navbars, footers)


routeTree.gen.ts
TanStack auto-generates this by scanning your routes/ folder. Never touch it.


src/routes/index.tsx 
a React component that represents the homepage (/). This is where your entire
PaperTrail UI lives.


App.jsx?
There isn't one in this project. In classic React setups (Create React App),
App.jsx is the root component. Here, routes/__root.tsx plays that role — it wraps
every page. And routes/index.tsx is your main page component. TanStack Router
replaced the need for a separate App.jsx.


Shadcn and ui folder
shadcn gives you <Button>, <Tabs>, <Dialog>, <Card> etc. — pre-styled, accessible,
ready to use. The key difference from other UI libraries: the code lives in your
project, inside src/components/ui/. You can open src/components/ui/button.tsx and
change how every button looks across the whole app.

<button className="bg-purple-600 px-4 py-2 rounded-lg text-white hover:bg-purple-700">
  Click me
</button>

// you write:
<Button variant="default">Click me</Button> and go to ui folder and button to modify it


The lib/ folder
lib/ is for non-UI code — utilities, API connections, data helpers. 
Nothing in here renders anything on screen. Think of it as the "brain" folder
vs the "face" folder.

groq.ts — connects to Groq API. Pure logic, no UI.

demo-data.ts — hardcoded strings and objects used as placeholder content
 until real AI responses come in.

utils.ts — one tiny function called cn() that merges Tailwind class names 
cleanly. shadcn uses it everywhere.

ALLORIGINS CONCEPT :
The CORS problem is a browser security rule. When your React app (running at localhost:8080)
tries to fetch from ar5iv.org, the browser checks:

"Does ar5iv.org have a header saying it allows requests from localhost:8080?"

ar5iv says no → browser blocks it. This isn't ar5iv being mean — it's a default 
security policy. Servers have to explicitly opt in to allow cross-origin requests.
allorigins is a middleman:

Without proxy:
Browser → ar5iv.org ✗ BLOCKED

With allorigins:
Browser → allorigins.win → ar5iv.org ✓ (server to server, no CORS rules)
         ← allorigins wraps response in JSON ←
         ← browser receives it fine ←

Server-to-server requests don't have CORS restrictions — that's a browser-only rule.
allorigins is just a tiny server that fetches any URL you give it and returns the
content wrapped in JSON: ( thats whay res.json() not res.text()  and then data.contents to extract data )
